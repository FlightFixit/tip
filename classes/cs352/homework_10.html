You are to provide the following for one user interface feature for your project. Please ensure that the chosen feature is not the same as your partner's.

  1. Please provide a detailed description (e.g. manual/tutorial) on the feature. Include the purpose of the feature, how it is to be used for its intended purpose, and any required syntax.

    Within our interface, one of the views the user has available is called the "orchestration interface." The function of the interface is the coordination and manipulation of multiple unmanned vehicles in response to a disaster situation. The orchestration view gives the user a two-dimensional map of the area where the robots are operating and allows selecting and guiding them. For example, the map might look like:

    The interface allows overlays of important information to be added. For example, one overlay is areas where other response personnel have requested samples be taken. When this overlay is activated, the previous example would look like:

    Within the orchestration interface robots are represented by the following image:

    The arrow represents the front of the robot. At any given time there is an active robot that will receive any new commands. The color filling the robot is purple and all other robots are filled with red.

    There are three tasks that it is possible to give a robot:

    • Waypoint: A position on the map that the robot should drive to. The robot will drive in a straight line from its current position to the next waypoint. Multiple waypoints may be specified and will be executed in sequence.
    • Collect Chemical Sample: A position on the map that the robot should drive to and collect a chemical sample.
    • Collect Dirt Sample: A position on the map that the robot should drive to and collect a dirt sample.

    Tasks are created by right-clicking on the map and selecting Add Task from the menu that comes up. From the associated dialog, the user may select which robot to create a task for (the active robot is the default) and may select the type of task to create.

    Tasks within the interface are represented by small circles of different colors: yellow for waypoint, green for chemical samples and brown for dirt samples. Lines are drawn between the tasks in the order they will be executed representing the path the robot will follow while executing the tasks.

  2. Please describe how you would create a tutorial for this feature. The tutorial may be an interactive tutorial or a demonstration. What would be included in the tutorial and how.

    I would do an interactive tutorial. The user would be instructed on how to activate the requested chemical samples interface and then stepped through the process for creating waypoints to guide the robot into position and collecting a sample.

  3. Please describe the necessary context-sensitive help, including the type of context-sensitive help to be provided.

    Context sensitive help is useful in the map for providing more detail about certain agents and tasks. When a user hovers over a robot more extensive information about the robot and its currently assigned tasks is shown. Also when hovering over a task, the interface could, but does not currently, show information about the robot the task is assigned to and the expected time to completion.
