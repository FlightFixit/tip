  1. Meeting #1: 2008/01/15

    This course is not focused on kinematics and manipulators. Rather we are looking for courses that incorporate mobile robots, their dynamics (but not as in-depth as a manipulator based kinematics course would cover), sensors and their use, path planning, behavior based robotics, etc. Compile a list of websites for the first meeting.

    Survey of Courses on Robotics

    The function of this study is to produce:

    • Course description
    • Primary and recommended readings
    • Syllabus
    • 15 week schedule
    • Reasonable milestones for project deliverables

    The students taking this course will have the following characteristics:

    • Junior, senior undergraduate or first year graduate students
    • Computer science, computer engineering, and mechanical enginnering studetns
    • Data structures, software design, calculus, statistics, and linear algebra experience

    Skills developed by this class will be:

    • Locomotion
    • Sensing
    • Sensor fusion
    • Rudimentary planning
    • Rudimentary manipulation

    A potential structure for the projects is team-based where there are a fixed number of roles and team members alternate through the various roles for different deliverables.

    Begin designing a series of incrimental projects to build toward a final project and help a student develop the skills of:

    • Planning (potentially)
    • Vision Recognition

    The nature of the project is guided by the types of sensors avialable in the robotics platform used. One of the more attractive platforms for beginners is to use legos with the Handy Board from MIT. Types of sensors available to lego robots:

    • Included in the kit are:
      • Touch sensor: detect if it pressed, released or has been bumped
      • Uni-directional photodetector: gives illumination magnitude and is coupled with an LED. These has been used for close proximity ranging measurements
      • Microphone: gives the amplitude of ambient noise
      • Ultrasound detector: gives randing and motion measurements
    • Additional parts from LEGO:
      • Accelorometer
      • Color sensor
      • Compass
      • Gyroscope
      • Multi-directional Infrared Array
      • Servomotor with positional feedback
    • Parts developed by third-parties are:
      • Galvanic Skin Response
      • Cadmium Sulfide Light Sensor: with a different frequency sensitivity than the silicon-based LEGO sensor
      • Differential Light Sensor: to do rough source position analysis
      • Air Pressure

    Other Lego-based challenges have included:

    • Can collecting at Utrecht University. Robots are released from a fixed postition into a fixed arena with random obstacles. They must locate aluminum cans, and return them to a specified position. This program was somewhat different because it used the RCX brick included with the lego set as the controlling microprocessor. The RCX brick has been superceded by the NXT brick based around a 32-bit ARM AT91SAM7S256. The processor is driven using a bytecode-based language called Next Byte Codes (NBC) for which there is a high-level language called Not eXactly C (NXC).
    • Lego FIRST Competitions: Heald each year, these are a series of competitons in which 7,000 teams of middle school students from around the world are scored on their robot creation abilities. Competitions generally seem to involve obstacle avoidance and debris clearing.
    • Robofest is a similar to the FIRST competition but based on a wider variety of competitions and platform. Video is available from the 2007 event.
    • The Trinity College Firefighting competition is a collegiate event. An autonomous robot must search a model of a house, locate a flame and extinguish it. The fastest time wins. There are various divisions including walking robots and an expert class that allows the use of swarms. A trailer of the competition DVD is online.
    • The Robotic Competitions FAQ lists 106 different competitons throughout 2008.
    • XY Plotter

  9. Meeting #9: 2008/03/11

    One of the goals for the class project is to emphasize the development of programs in simulation and then transition those skills over to actual robot construction. So far as robotics simulators there are two primary options:

    • Player/Stage
      • Open source
      • Linux-based
      • Unstable API
    • Microsoft Robotics Studio
      • Newly released
      • Windows-based
      • Polished product

    In order to reduce the amount of additional information that the students, who operate in Microsoft Visual Studio for their other classes, the choice was made to use Robotics Studio. The question is then which platform to target.

    The introduction goes on a great deal about the REST style composable architecture and includes the sentence, "Remotely connected scenarios enable you to communicate from a PC to the robot through a serial port, Bluetooth®istered;, 802.11, or RF. Programs can also execute natively on PC-based robots running Microsoft Windows, enabling fully autonomous operation." An entry at Coding4Fun and MSDN seem to confirm that Robotics Studio requires Windows to run and integrates with different platforms only via remote control.
